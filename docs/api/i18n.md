# i18n 层 API 文档

> 命名空间：`error_system::i18n`

i18n 层提供错误码的多语言翻译能力，支持通过 JSON 字典文件将错误码字段映射为人类可读的本地化文本。

---

## language_t

**头文件**：`error_system/i18n/language.h`

支持的语言枚举，用于指定翻译器使用的目标语言。

```cpp
enum class language_t : uint8_t {
    zh_cn,  // 简体中文
    en_us,  // 英语（美国）
    _count
};

// 辅助函数
constexpr uint8_t to_int(language_t lang) noexcept;
constexpr language_t from_int(uint8_t lang) noexcept;
constexpr language_t from_string(std::string_view lang) noexcept;
constexpr const char* to_string(language_t lang) noexcept;
constexpr bool is_valid(uint8_t lang) noexcept;
```

---

## i_translator_t

**头文件**：`error_system/i18n/i_translator.h`

翻译器抽象接口，所有翻译器实现都必须继承此接口。

```cpp
class i_translator_t {
public:
    virtual ~i_translator_t() = default;

    // 将错误码翻译为当前语言文本
    virtual std::string translate(const core::error_code_t& code) const noexcept = 0;

    // 获取当前语言
    virtual language_t get_language() const noexcept = 0;

    // 切换语言
    virtual void set_language(language_t lang) noexcept = 0;

    // 获取翻译格式
    virtual const std::string& get_format() const noexcept = 0;

    // 设置翻译格式
    virtual void set_format(const std::string& format) noexcept = 0;
};
```

## json_translator_t

**头文件**：`error_system/i18n/json_translator.h`

基于 JSON 字典的翻译器实现，继承 `i_translator_t`。

### 字典格式

JSON 字典位于 `include/error_system/i18n/languages/<lang>.json`，结构如下：

```json
{
  "error_level": {
    "debug": "调试",
    "error": "错误",
    "fatal": "致命"
  },
  "domain": {
    "database": "数据库层",
    "network": "网络层"
  },
  "subsystem": {
    "database": {
      "mysql": "MySQL"
    }
  },
  "module": {
    "database": {
      "connection": "连接模块"
    }
  }
}
```

### 翻译输出格式

默认格式：
```
[Level: 错误, System: 数据库层, Subsystem: MySQL, Module: 连接模块] Code: 404
```

可通过 `set_format()` 自定义格式字符串，使用 `{}` 作为占位符。

### 实现细节

`translate()` 内部使用 `traits::resolve_subsystem_type()` / `resolve_subsystem()` / `resolve_module_type()` / `resolve_module()` 将子系统和模块编号解析为字符串，再查询 JSON 字典进行翻译。

### 使用示例

```cpp
#include "error_system/i18n/json_translator.h"

json_translator_t translator(language_t::zh_cn);
std::string msg = translator.translate(code);

// 切换语言
translator.set_language(language_t::en_us);

// 自定义输出格式
translator.set_format("[{}] {} | {} | {} | Code: {}");
```

## translator_registry_t

**头文件**：`error_system/i18n/translator_registry.h`

全局翻译器单例注册表，使 `error_context_t::to_string()` 无需每次传入翻译器。

> **注意**：不持有翻译器所有权，调用方需保证翻译器对象在使用期间始终有效。

### 方法

```cpp
// 获取单例
static translator_registry_t& instance() noexcept;

// 注册全局翻译器（传 nullptr 可清除）
void set(i_translator_t* translator) noexcept;

// 获取当前全局翻译器（未注册时自动创建默认翻译器，永不为 nullptr）
// 默认翻译器使用 error_config::get_default_language() 指定的语言
i_translator_t* get() const noexcept;

// 是否已由用户显式注册翻译器
bool has_translator() const noexcept;
```

### 默认语言配置

`translator_registry_t::get()` 在未显式注册翻译器时，会自动创建一个默认的 `json_translator_t`。默认语言由 `error_config` 控制：

```cpp
// 设置默认语言为英文
error_config::set_default_language("en_us");

// 首次调用 get() 时，自动创建英文翻译器
auto* translator = translator_registry_t::instance().get();
```

### 使用示例

```cpp
// 程序启动时注册一次
json_translator_t translator(language_t::zh_cn);
translator_registry_t::instance().set(&translator);

// 之后 error_context_t::to_string() 自动使用全局翻译器
error_context_t ctx{code, "连接超时"};
std::cout << ctx.to_string();  // 自动翻译

// 临时覆盖语言
json_translator_t en(language_t::en_us);
std::cout << ctx.to_string(&en);  // 使用英文
```
